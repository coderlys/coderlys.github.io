    认识web框架

    • 前面我们已经学习了使用http内置模块来搭建Web服务器，为什么还要使用框架？
      • 原生http在进行很多处理时，会较为复杂；
      • 有URL判断、Method判断、参数处理、逻辑代码处理等，都需要我们自己来处理和封装；
      • 并且所有的内容都放在一起，会非常的混乱；
    • 目前在Node中比较流行的Web服务器框架是express、koa；
      • 我们先来学习express，后面再学习koa，并且对他们进行对比；
    • express早于koa出现，并且在Node社区中迅速流行起来：
      • 我们可以基于express快速、方便的开发自己的Web服务器；
      • 并且可以通过一些实用工具和中间件来扩展自己功能；

    Express安装

    • express的使用过程有两种方式

      • 方式一：通过express提供的脚手架，直接创建一个应用的骨架；
      • 方式二：从零搭建自己的express应用结构；

    • 方式一：安装express-generator

      安装脚手架

    # 1.1：Express基本使用

    • 我们来创建第一个express项目
      • 我们会发现，之后的开发过程中，可以方便的将请求进行分离；
      • 无论是不同的URL，还是get、post等请求方式；
      • 这样的方式非常方便我们已经进行维护、扩展；
    • 请求的路径中如果有一些参数，可以这样表达；

      • 在request对象中要获取可以通过 req.params.userId;

    # 1.2：认识中间件

    • Express是一个路由和中间件的web框架，它本身功能非常少

      • Express应用程序本质是一系列中间件函数的调用；

    • 中间件是什么呢？

      • 中间件的本质传递给express的一个回调函数
      • 这个回调函数接受三个参数：
        • 请求对象(request对象)；
        • 响应对象(response对象)；
        • next函数(在express中定义的用于执行下一个中间件的函数)；

    # 1.2.1：应用中间件-自己编写

    • 那么，如何将一个中间件应用到我们的应用程序中呢？
      • express主要提供了两种方式：app/router.use和app/router.methods；
      • 可以是 app，也可以是router，router我们后续再学习:
      • methods指的是常用的请求方式，比如： app.get或app.post等；
    • 我们先来学习use的用法，因为methods的方式本质是use的特殊情况；
      • 案例一：最普通的中间件
      • 案例二：path匹配中间件
      • 案例三：path和method匹配中间件
      • 案例四：注册多个中间件
    const express=require("express");
let app=express();
//http://127.0.0.1:8000/abc ,不管后面传啥数据页，都会执行以下代码并输出内容
app.use((req,res,next)=>{
    console.log("use1"); //use1
    next()
})
app.use((req,res,next)=>{
    console.log("use2"); //use2
    next()
})
app.use((req,res,next)=>{
    console.log("use3"); // use3
    res.end("12")
})
app.listen("8000",()=>{
    console.log("服务器已开启");
})

    # 1.3：客户端发送请求的方式

    1.2.2：应用中间件-body解析

    • 在客户端发送post请求时，会将数据放到body中：
      • 客户端可以通过json的方式传递；
      • 也可以通过form表单的方式传递；

    # 1.3.1：JSON的方式提交数据

    自己从零去编写编写解析request body中间件

    const app=express();
app.use((req,res,next)=>{
    if(req.headers["content-type"]==="application/json"){
        req.on("data",(data)=>{
            let da= JSON.parse(data.toString())
                req.body=da;
        })
        req.on("end",()=>{
            next()
        })
    }else{
        next()
    } 
}),
app.get("/login",(req,res,next)=>{
    console.log(req.body); // { username: 'lys', password: '888888' }
    res.end("登录成功")
}),

app.listen(8000,()=>{
    console.log("服务器请求成功");
})

    # 1.3.3：form-data的方式提交数据

    const path=require("path")

const express=require("express");
const multer=require("multer"); //使用第三方库 https://github.com/expressjs/multer/blob/master/doc/README-zh-cn.md

let storage=multer.diskStorage({
    destination:function(req,file,cb){ // 来确定上传的文件应该存储在哪个文件夹
        cb(null,'./uploads')
    },
    filename:function(req,file,cb){ // 来确定文件夹中的文件名的确定
        cb(null,Date.now()+path.extname(file.originalname))
    }
})
let upload=multer({ storage })

const app=express();
app.get("/login",upload.any(),(req,res,next)=>{
    console.log(req.body); // 提交一些简单的数据， [Object: null prototype] { name: 'lys' }
    res.end("登录成功")
}),
app.get("/upload",upload.fields([{name:'file',maxCount:2},{name:'file3',maxCount:1}]),(req,res,next)=>{ 
    // 提交图片文件
    console.log(req.files);  //打印图片的信息
    res.end("上传成功")
})
app.listen(8000,()=>{
    console.log("服务器请求成功");
})

    # 1.3.4：params传递参数

    const express =require("express");
let app=express();
// 请求地址：http://127.0.0.1:8000/login/2/lys
app.get("/login/:id/:name",(req,res,next)=>{

    # 1.5：请求数据日志记录

    如果我们希望将请求日志记录下来，那么可以使用express官网开发的第三方库：morgan

    const express=require("express");
const fs=require("fs")
const morgan=require('morgan');

const loggerWriter=fs.createWriteStream('./log/access.log',{flags:'a+'})
 
let app=express();
app.use(morgan('combined',{stream:loggerWriter}));

app.get("/login",(req,res,next)=>{
   res.end("记录成功")

    1.6：Express的路由

    • 如果我们将所有代码逻辑都写在app中，那么app会变得越来越复杂；

      • 一方面完整的web服务器包含非常多的处理逻辑
      • 另一方面有些处理逻辑其实是一个整体，我们应该将它们放在一起，比如对users相关处理

    • 我们可以使用express.router来创建一个路由处理程序；

      • 一个Router实例拥有完整的的中间件和路由系统；
      • 因此，它也称为迷你应用程序（mini-app）;
      const express=require("express");
const userRouter=express.Router();
userRouter.get("/",(req,res,next)=>{
    res.end("用户列表")
})
userRouter.post("/",(req,res,next)=>{
    res.end("创建用户")
})
userRouter.delete("/",(req,res,next)=>{
    res.end("删除用户")
})
module.exports=userRouter

    # 1.7：服务端的错误处理

    const express =require("express");
let app=express();

 app.post("/login",(req,res,next)=>{
    const isLogin=false;
    if(isLogin){
        res.json("user login success~")
    }else{
        next(new Error ("USERNAME_DOES_NOT_EXISTS"))
    }
 })
 app.get("/register",(req,res,next)=>{
     const isRegister=false;
     if(isRegister){
         res.json("user register success~")
     }else{
         next(new Error("USERNAME_ALREADY_EXISTS"))
     }
 })

 app.use((err,req,res,next)=>{
    let message="" ;
    let status=400;
   switch(err.message){
       case "USERNAME_DOES_NOT_EXISTS":
             message="username does not exists~";
       break;
 
       case "USERNAME_ALREADY_EXISTS":
           message="username_already_exists";
        break;
        
       default:
            message="NOT FOUND"
   }

   res.status(status)
   res.json({
       errCode:status,
       errMessage:message
   })
 })
app.listen(8000,()=>{
    console.log("服务器已启动");
})
